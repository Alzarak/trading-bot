# External Integrations

**Analysis Date:** 2026-03-22

## APIs & External Services

**Alpaca Markets (primary trading API):**
- Service: Alpaca Markets — US stock brokerage with REST API
- SDK/Client: `alpaca-py==0.43.2`
- Auth: `ALPACA_API_KEY` + `ALPACA_SECRET_KEY` environment variables
- Clients instantiated in `scripts/bot.py` via `create_clients()`
- Two client types:
  - `alpaca.trading.client.TradingClient` — order submission, account info, positions, market clock
  - `alpaca.data.historical.StockHistoricalDataClient` — OHLCV bar fetching
- Paper trading toggle: `paper=True/False` on `TradingClient` constructor, controlled by `ALPACA_PAPER` env var
- Market data feed: IEX (free tier) — `feed="iex"` in `StockBarsRequest`
- Timeframe: 1-minute bars (`TimeFrame.Minute`)

**Alpaca MCP Server (optional, opt-in):**
- Service: Official Alpaca MCP server — 44 real-time market data tools
- Install: `uvx alpaca-mcp-server` (separate process via MCP protocol)
- Auth: `ALPACA_PAPER_TRADE` env var (independent of bot's `ALPACA_PAPER`)
- Configured via: `claude mcp add` during `/initialize` if user opts in
- Config stored in: `.mcp.json` (gitignored — contains API key references)

## Data Storage

**Databases:**
- SQLite — primary persistence layer
  - Location: `{CLAUDE_PLUGIN_DATA}/trading.db`
  - Client: `sqlite3` stdlib (no ORM)
  - WAL journal mode enabled for crash robustness
  - Managed by: `scripts/state_store.py` — `StateStore` class

  **Schema (4 tables):**
  - `positions` — open/closed positions (symbol PK, qty, entry_price, stop_price, strategy, status)
  - `orders` — order records (client_order_id UUID, alpaca_order_id, side, qty, status)
  - `trade_log` — immutable trade history (action, price, qty, pnl, strategy, order_type)
  - `day_trades` — PDT tracking (symbol, date) for rolling 7-calendar-day window

**File Storage:**
- `{CLAUDE_PLUGIN_DATA}/audit/claude_decisions.ndjson` — NDJSON audit trail of all Claude recommendations and execution outcomes (managed by `scripts/audit_logger.py`)
- `{CLAUDE_PLUGIN_DATA}/circuit_breaker.flag` — presence triggers trading halt, manually deleted to resume
- `{CLAUDE_PLUGIN_DATA}/pdt_trades.json` — legacy PDT tracking (migrated to SQLite on first startup)
- `{CLAUDE_PLUGIN_DATA}/config.json` — runtime configuration generated by `/initialize`
- `{CLAUDE_PLUGIN_DATA}/requirements.txt.sha256` — hash for dependency change detection
- `{CLAUDE_PLUGIN_DATA}/venv/` — isolated Python virtualenv

**Caching:**
- None — all state persisted to SQLite or flat files

## Claude AI Integration

**Role:** Market analyst only — Claude generates trade recommendations but never submits orders directly.

**Integration pattern:**
1. `scripts/market_scanner.py` (`MarketScanner`) fetches OHLCV bars + computes indicators via pandas-ta
2. `scripts/claude_analyzer.py` (`ClaudeAnalyzer.build_analysis_prompt()`) formats indicator DataFrames into structured prompts
3. Claude (via `market-analyst` agent) receives prompt and returns JSON recommendation
4. `ClaudeAnalyzer.parse_response()` validates JSON, applies confidence threshold (default 0.6)
5. Valid `ClaudeRecommendation` converts to `Signal` via `to_signal()` (defined in `scripts/types.py`)
6. Signal routes through `scripts/order_executor.py` (`OrderExecutor`) — all 4 risk checks enforced
7. `scripts/audit_logger.py` (`AuditLogger`) logs every recommendation + execution outcome to NDJSON

**Agent definitions:**
- `agents/market-analyst.md` — model: sonnet, tools: Read/Bash/Glob/Grep
- `agents/risk-manager.md` — model: haiku, tools: Read/Bash/Grep
- `agents/trade-executor.md` — model: haiku, tools: Read/Bash/Grep

## Claude Code Hooks (Lifecycle Integration)

**SessionStart:**
- Hook: `bash "${CLAUDE_PLUGIN_ROOT}/scripts/install-deps.sh"` (timeout: 120s)
- Purpose: Install Python dependencies on session start if `requirements.txt` changed

**PreToolUse (Bash tool intercept):**
- Hook: `bash "${CLAUDE_PLUGIN_ROOT}/hooks/validate-order.sh"` (timeout: 10s)
- Matches: `Bash` tool calls
- Guards: Intercepts commands matching `submit_order|place_order|execute_trade|bot.py.*--(order|trade|submit)`
- Checks circuit breaker flag and PDT count (SQLite primary, JSON fallback)
- Denies with structured JSON if either check fails

**Stop:**
- Hook: `bash "${CLAUDE_PLUGIN_ROOT}/hooks/check-session.sh"` (timeout: 10s)
- Purpose: Warns about open positions and active circuit breaker at session end
- Always allows stop (never blocks) — advisory only
