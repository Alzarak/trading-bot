# Quick Task 260322-18l: Fix PreToolUse Bash Hook Error (validate-order.sh) Summary

**One-liner:** Added PLUGIN_ROOT dirname fallback to validate-order.sh so the PreToolUse hook resolves paths correctly in dev mode when CLAUDE_PLUGIN_ROOT is unset.

## What Was Done

Applied the identical dual-context header pattern from `scripts/install-deps.sh` to `hooks/validate-order.sh`.

The original script had two issues:
1. `DATA_DIR="${CLAUDE_PLUGIN_DATA:-/tmp}"` — fell back to `/tmp` in dev mode (inconsistent with install-deps.sh)
2. `CB_FLAG="${CLAUDE_PLUGIN_DATA:-/tmp}/circuit_breaker.flag"` — duplicated the fallback inline (instead of using DATA_DIR)

## Changes Made

**hooks/validate-order.sh** — Path resolution header added after `set -euo pipefail`:
- `PLUGIN_ROOT="${CLAUDE_PLUGIN_ROOT:-$(cd "$(dirname "$0")/.." && pwd)}"` — resolves to repo root in dev mode (script lives in `hooks/`, so `..` is correct)
- `DATA_DIR="${CLAUDE_PLUGIN_DATA:-${PLUGIN_ROOT}/.plugin-data}"` — falls back to `.plugin-data/` in dev mode
- `CB_FLAG="${DATA_DIR}/circuit_breaker.flag"` — uses DATA_DIR (no duplicate fallback)
- All path declarations moved before `INPUT=$(cat /dev/stdin)` — clean execution order
