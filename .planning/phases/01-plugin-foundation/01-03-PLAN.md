  <action>
Create `commands/initialize.md` -- the interactive setup wizard prompt. Remove `commands/.gitkeep` if it exists.

The file must have YAML frontmatter followed by the wizard prompt. Keep the file UNDER 200 lines total (per research pitfall #6 -- move strategy details to references/).

Frontmatter fields (use standard YAML frontmatter delimiters):
- name: initialize
- description: "Interactive setup wizard -- configure the trading bot preferences and generate config.json"
- argument-hint: "[--reset]"
- allowed-tools: AskUserQuestion, Write, Bash, Read

The wizard prompt body must instruct Claude to execute these steps:

**Pre-check:** Use Bash to check if `${CLAUDE_PLUGIN_DATA}/config.json` exists. If it does and user did NOT pass `--reset`, ask if they want to reconfigure or keep existing. If `--reset` was passed, proceed with fresh setup.

**Step 1 - Experience Level (CMD-02):**
Ask: "What is your trading experience level?"
Options: 1) Beginner (new to trading, prefer guided defaults), 2) Intermediate (some experience, balanced options), 3) Expert (understand trading well, show all parameters)
Store as `experience_level`. This determines the flow for all subsequent steps.

**Step 2 - Risk Tolerance (CMD-03):**
Beginner: Present with explanations, default to "conservative", options: conservative (recommended), moderate.
Intermediate: All three options (conservative, moderate, aggressive), no default.
Expert: All three options with technical context, no default.
Store as `risk_tolerance`.

**Step 3 - Budget and Trading Mode (CMD-03, ALP-02, ALP-03):**
Ask for budget in USD (starting capital to allocate).
Beginner: Paper trading ONLY (inform user, no choice). Set paper_trading=true.
Intermediate/Expert: Ask paper vs live. Warn that live uses real money.
Store budget_usd and paper_trading.
If live mode selected, prompt for Alpaca API keys immediately.

**Step 4 - Strategies (CMD-12):**
Read `references/trading-strategies.md` for descriptions.
Present all four: momentum, mean_reversion, breakout, vwap.
Beginner: Recommend momentum only, allow selecting 1-2.
Expert: Allow selecting any combination with custom weights.
For each selected strategy, use default parameters from the reference file.
Store as strategies array with {name, weight, params} objects.

**Step 5 - Autonomy Mode (CMD-04):**
Beginner: Default to fixed_params with brief explanation.
Expert: Present both options:
  - fixed_params: Bot uses exact risk parameters you set
  - claude_decides: Claude adjusts position size and entry thresholds per-trade within bounds
Store as autonomy_mode.

**Step 6 - Market Hours and Watchlist (CMD-03):**
Market hours: Default market_hours_only=true. Expert can toggle to false.
Watchlist: Ask for ticker symbols (comma-separated).
Beginner: Suggest defaults: AAPL, MSFT, GOOGL, AMZN, SPY.
Expert: No defaults, freeform input.
Validate tickers are uppercase alpha strings.

**Final Step - Write Config (CMD-05, STATE-03):**
Use Bash to resolve ${CLAUDE_PLUGIN_DATA} path: `echo "${CLAUDE_PLUGIN_DATA}"`
Set computed defaults based on risk_tolerance:
  - conservative: max_position_pct=5.0, max_daily_loss_pct=2.0
  - moderate: max_position_pct=10.0, max_daily_loss_pct=3.0
  - aggressive: max_position_pct=15.0, max_daily_loss_pct=5.0
Set autonomy_level to "notify_only" for beginners, "full_auto" for experts, "semi_auto" for intermediate.
Set config_version to "1".
Add created_at with ISO 8601 timestamp.
Write complete config.json to ${CLAUDE_PLUGIN_DATA}/config.json using Bash heredoc.
Write .env template to ${CLAUDE_PLUGIN_DATA}/.env with:
  ALPACA_API_KEY=your_key_here (or user-provided key if live mode)
  ALPACA_SECRET_KEY=your_secret_here (or user-provided key if live mode)
  ALPACA_PAPER=true (or false if live mode)
Display a summary table of the configuration.
Tell user where files were written and what to do next.

**Constraints in the prompt:**
- Instruct Claude to re-prompt on invalid input with explanation of valid options
- Instruct Claude to use AskUserQuestion for each step
- Reference references/trading-strategies.md by path for strategy details
- Never store API keys in config.json -- .env only
- Use Bash to resolve ${CLAUDE_PLUGIN_DATA} before writing (avoid expansion ambiguity)

After creating the file, run: `rm -f commands/.gitkeep`
