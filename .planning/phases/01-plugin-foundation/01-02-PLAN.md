- name: trading-rules
- description: "Core trading rules and risk parameters for the trading bot. Provides position sizing limits, market hours constraints, PDT rules, strategy context, and Alpaca API conventions. Auto-load when the conversation involves trading decisions, risk management, strategy execution, order placement, or market analysis."

Body must contain these sections with the following content:

Section "## Core Trading Rules" - intro stating rules apply to ALL trading operations.

Section "### Market Hours":
- US stock market hours: 9:30 AM - 4:00 PM Eastern Time
- Pre-market: 4:00 AM - 9:30 AM ET (not supported in v1)
- After-hours: 4:00 PM - 8:00 PM ET (not supported in v1)
- Use `zoneinfo.ZoneInfo("America/New_York")` for timezone (stdlib, no install)
- Always check Alpaca market clock before submitting orders

Section "### Position Sizing":
- Position size = `max_position_pct` of account equity (from config.json)
- Default: 5% per position (conservative), 10% (moderate), 15% (aggressive)
- Maximum concurrent positions: 10 (hard limit, configurable down)
- Never risk more than `max_daily_loss_pct` of portfolio in a single day

Section "### Pattern Day Trader (PDT) Rule":
- Accounts under $25,000: max 3 day trades per rolling 5-business-day window
- A day trade = buying and selling same security on same day
- Track count in SQLite state; block new entries at count >= 3
- PDT applies to both paper and live accounts on Alpaca

Section "### Risk Controls":
- Circuit breaker: halt all trading when daily loss exceeds `max_daily_loss_pct`
- Every order MUST have a stop-loss (bracket order or explicit stop order)
- ATR-based stops: stop distance = ATR * multiplier (default 1.5x)
- No averaging down into losing positions

Section "### Alpaca API Conventions":
- Use `alpaca-py` SDK (not deprecated `alpaca-trade-api`)
- `TradingClient(api_key, secret_key, paper=True)` for paper trading
- `TradingClient(api_key, secret_key, paper=False)` for live trading
- API keys loaded from environment variables via pydantic-settings, never hardcoded
- All API calls wrapped with exponential backoff retry

Section "### Strategy Framework":
- Available strategies: momentum, mean_reversion, breakout, vwap
- Each strategy has independent parameters in config.json
- Strategies produce signals: BUY, SELL, or HOLD with confidence 0.0-1.0
- Multiple strategies can run simultaneously with configurable weights

Section "### Claude's Role":
- Claude is a strategy-level analyst ONLY
- Claude analyzes opportunities and returns structured JSON recommendations
- Claude NEVER submits orders directly
- All Claude recommendations pass through the deterministic Python risk manager
- Every Claude decision is audit-logged with full reasoning

**File 2: `references/trading-strategies.md`**

Title: "# Trading Strategies Reference"

Four strategy sections, each with: Signal logic, Entry conditions, Exit conditions, Default parameters, Config name.

Momentum Strategy (config name: `momentum`):
- Entry: RSI crosses above 30 AND MACD histogram turns positive, EMA 9 crosses above EMA 21, Volume above 20-period average
- Exit: RSI crosses above 70 OR MACD histogram turns negative, EMA 9 crosses below EMA 21, stop-loss hit
- Params: rsi_period=14, macd_fast=12, macd_slow=26, macd_signal=9, ema_short=9, ema_long=21

Mean Reversion Strategy (config name: `mean_reversion`):
- Entry (long): Price touches lower Bollinger Band (2 std dev), RSI below 30, price within 2% of band
- Exit: Price returns to middle Bollinger Band (SMA 20), stop-loss hit
- Params: bb_period=20, bb_std_dev=2.0, rsi_period=14, rsi_oversold=30, rsi_overbought=70

Breakout Strategy (config name: `breakout`):
- Entry: Price breaks above 20-period high on volume > 1.5x average, ATR expanding, MACD positive
- Exit: Price falls below breakout level, volume drops on 3 bars, stop-loss hit
- Params: lookback_period=20, volume_multiplier=1.5, atr_period=14

VWAP Reversion Strategy (config name: `vwap`):
- Entry (long): Price drops > 1.5% below VWAP, RSI below 40, time 10AM-3PM ET
- Exit: Price returns to VWAP, or moves 2% further (stop)
- Params: deviation_threshold_pct=1.5, rsi_period=14, max_deviation_pct=2.0, trading_start_hour=10, trading_end_hour=15

Separate each strategy section with a horizontal rule.

**File 3: `references/risk-rules.md`**

Title: "# Risk Rules Reference"

Sections:
- Circuit Breaker: trigger on max_daily_loss_pct exceeded, halt all orders, manual restart required, defaults conservative=2%, moderate=3%, aggressive=5%, formula: (start_equity - current_equity) / start_equity * 100
- Position Sizing: formula position_value = equity * (max_position_pct/100), shares = floor(value/price), reject if shares < 1, budget cap, defaults conservative=5%, moderate=10%, aggressive=15%
- Stop-Loss Rules: every order MUST have stop-loss, ATR-based: stop_price = entry - (ATR * multiplier), default multiplier 1.5x conservative / 1.0x aggressive, minimum 0.5% of entry, trailing stops trail by ATR * multiplier
- PDT Protection: 3 day trades per 5 business days under $25K, warn at 2, block at 3, tracked in SQLite
- Maximum Position Count: default 10, configurable via max_positions
- Order Safety: idempotency via client_order_id UUID, check status before retry on failure, ghost position prevention, exponential backoff 1s/2s/4s/8s max 4 retries
- Autonomy Modes: fixed_params (exact config values), claude_decides (position size 50-150% of configured, entry threshold can tighten not loosen, stop-loss can widen not tighten, reasoning required)
