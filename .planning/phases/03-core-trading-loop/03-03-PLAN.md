<objective>
Build the SQLite-backed StateStore for persisting positions, orders, trade history, and day trades, with crash recovery via Alpaca position reconciliation.

Purpose: State persistence is the safety substrate. Without it, a crash means lost position tracking, incorrect PDT counts, and no trade audit trail. The StateStore also provides the migration path from pdt_trades.json to SQLite.
Output: `scripts/state_store.py` (StateStore class), `tests/test_state_store.py` (unit tests including crash recovery scenarios)
</objective>

<!-- SQLite schema from research -->
```sql
CREATE TABLE IF NOT EXISTS positions (
    symbol TEXT PRIMARY KEY, qty INTEGER NOT NULL, entry_price REAL NOT NULL,
    stop_price REAL NOT NULL, strategy TEXT NOT NULL, opened_at TEXT NOT NULL,
    alpaca_order_id TEXT, status TEXT NOT NULL DEFAULT 'open'
);
CREATE TABLE IF NOT EXISTS orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT, client_order_id TEXT UNIQUE NOT NULL,
    alpaca_order_id TEXT, symbol TEXT NOT NULL, side TEXT NOT NULL,
    order_type TEXT NOT NULL, qty INTEGER NOT NULL, limit_price REAL,
    stop_price REAL, status TEXT NOT NULL, submitted_at TEXT NOT NULL, filled_at TEXT
);
CREATE TABLE IF NOT EXISTS trade_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT, symbol TEXT NOT NULL, action TEXT NOT NULL,
    price REAL NOT NULL, qty INTEGER NOT NULL, pnl REAL, strategy TEXT NOT NULL,
    order_type TEXT NOT NULL, logged_at TEXT NOT NULL
);
CREATE TABLE IF NOT EXISTS day_trades (
    id INTEGER PRIMARY KEY AUTOINCREMENT, symbol TEXT NOT NULL, date TEXT NOT NULL
);
```

Create `scripts/state_store.py` with a `StateStore` class:

1. `__init__(self, db_path: str | Path)` — connects to SQLite at `db_path`. Sets `journal_mode=WAL` for robustness (survives power loss). Sets `check_same_thread=False` for future thread safety. Calls `_create_tables()`. Also checks for and migrates `pdt_trades.json` if it exists in the same directory as `db_path`.

2. `_create_tables(self)` — executes the 4 CREATE TABLE IF NOT EXISTS statements from the schema above. Uses `self.conn.executescript()`.

3. **Positions CRUD:**
   - `upsert_position(self, symbol, qty, entry_price, stop_price, strategy, opened_at=None, alpaca_order_id=None)` — INSERT OR REPLACE into positions. `opened_at` defaults to `datetime.now().isoformat()`. Status defaults to 'open'.
   - `get_open_positions(self) -> dict[str, dict]` — SELECT * WHERE status='open'. Returns dict keyed by symbol.
   - `mark_position_closed(self, symbol: str)` — UPDATE status='closed' WHERE symbol=symbol.
   - `get_position(self, symbol: str) -> dict | None` — SELECT single position by symbol.

4. **Orders CRUD:**
   - `record_order(self, client_order_id, alpaca_order_id, symbol, side, order_type, qty, status, limit_price=None, stop_price=None)` — INSERT into orders. `submitted_at` = now ISO.
   - `update_order_status(self, client_order_id: str, status: str, filled_at: str | None = None)` — UPDATE.
   - `get_order(self, client_order_id: str) -> dict | None` — SELECT single order.

5. **Trade log:**
   - `log_trade(self, symbol, action, price, qty, strategy, order_type, pnl=None)` — INSERT into trade_log. `logged_at` = now ISO.
   - `get_trade_history(self, limit: int = 100) -> list[dict]` — SELECT recent trades ORDER BY logged_at DESC LIMIT limit.

6. **Day trades (PDT):**
   - `record_day_trade(self, symbol: str, date: str)` — INSERT into day_trades.
   - `get_day_trade_count(self, window_start: str) -> int` — SELECT COUNT(*) WHERE date > window_start.
   - `get_day_trades(self, window_start: str) -> list[dict]` — SELECT all day trades after window_start.

7. **Crash recovery:**
   - `reconcile_positions(self, trading_client) -> dict` — implements the 3-case reconciliation:
     - Case 1: Position in Alpaca but NOT in SQLite -> upsert with strategy="unknown_post_crash", stop_price=0.0, log warning
     - Case 2: Position in SQLite as 'open' but NOT in Alpaca -> mark_position_closed, log warning
     - Case 3: Position in both -> update qty and entry_price from Alpaca (source of truth)
     - Returns a summary dict: `{"inserted": [...], "closed": [...], "updated": [...]}`

8. **PDT migration:**
   - `_migrate_pdt_json(self, json_path: Path)` — reads pdt_trades.json, inserts each entry into day_trades table, renames json file to `pdt_trades.json.migrated`. Called once from __init__ if the file exists. Uses loguru to log the migration.

9. `close(self)` — closes the SQLite connection.

Use loguru for all logging. Use `from pathlib import Path` for file paths. Use `from datetime import datetime` for timestamps.
